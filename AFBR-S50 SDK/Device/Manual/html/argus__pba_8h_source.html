14 /*!***************************************************************************
15  * @defgroup arguspba Pixel Binning Algorithm
16  * @ingroup argusapi
17  *
18  * @brief Pixel Binning Algorithm (PBA) parameter definitions and API functions.
19  *
20  * @details Defines the generic pixel binning algorithm (PBA) setup parameters
21  * and data structure.
22  *
23  * The PBA module contains filter algorithms that determine the
24  * pixels with the best signal quality and extract an 1d distance
25  * information from the filtered pixels.
26  *
27  * The pixel filter algorithm is a three-stage filter with a
28  * fallback value:
29  *
30  * -# A fixed pre-filter mask is applied to statically disable
31  * specified pixels.
32  * -# A relative and absolute amplitude filter is applied in the
33  * second stage. The relative filter is determined by a ratio
34  * of the maximum amplitude off all available (i.e. not filtered
35  * in stage 1) pixels. Pixels that have an amplitude below the
36  * relative threshold are dismissed. The same holds true for
37  * the absolute amplitude threshold. All pixel with smaller
38  * amplitude are dismissed.\n
39  * The relative threshold is useful to setup a distance
40  * measurement scenario. All well illuminated pixels are
41  * selected and considered for the final 1d distance. The
42  * absolute threshold is used to dismiss pixels that are below
43  * the noise level. The latter would be considered for the 1d
44  * result if the maximum amplitude is already very low.
45  * -# A distance filter is used to distinguish pixels that target
46  * the actual object from pixels that see the brighter background,
47  * e.g. white walls. Thus, the pixel with the minimum distance
48  * is referenced and all pixel that have a distance between
49  * the minimum and the given minimum distance scope are selected
50  * for the 1d distance result. The minimum distance scope is
51  * determined by an relative (to the current minimum distance)
52  * and an absolute value. The larger scope value is the
53  * relevant one, i.e. the relative distance scope can be used
54  * to heed the increasing noise at larger distances.
55  * -# If all of the above filters fail to determine a single valid
56  * pixel, the golden pixel is used as a fallback value. The
57  * golden pixel is the pixel that sits right at the focus point
58  * of the optics at large distances.
59  * .
60  *
61  * After filtering is done, there may be more than a single pixel
62  * left to determine the 1d signal. Therefore several averaging
63  * methods are implemented to obtain the best 1d result from many
64  * pixels. See #argus_pba_averaging_mode_t for details.
65  *
66  *
67  * @addtogroup arguspba
68  * @{
